# Editorial Infographic

Modern magazine-style editorial infographic with clear visual storytelling and professional polish.

---

## 1. Visual Theme & Atmosphere

- **Brand Personality:** Authoritative, clear, sophisticated, and intellectually engaging
- **Emotional Description:** High-quality magazine explainer aesthetic that transforms complex information into digestible narratives
- **Key Characteristics:**
  - Clean illustrations and structured layouts
  - Professional typography with editorial sophistication
  - Visual metaphors for abstract concepts
  - Information-dense but organized with clear hierarchy
  - Think Wired, The Verge, or high-end science publications
  - Magazine-quality polish throughout

---

## 2. Color Palette & Roles

| Role | Color | Hex | Usage |
|------|-------|-----|-------|
| **Primary** | Pure White | #FFFFFF | Primary background |
| **Secondary** | Light Gray | #F8F9FA | Section backgrounds, alternating rows |
| **Primary Text** | Near Black | #1A1A1A | Headlines, body text |
| **Secondary Text** | Dark Gray | #4A5568 | Captions, metadata, annotations |
| **Accent 1** | Editorial Blue | #2563EB | Primary accent, links, CTAs |
| **Accent 2** | Coral | #F97316 | Secondary accent, highlights |
| **Accent 3** | Emerald | #10B981 | Positive elements, success states |
| **Accent 4** | Amber | #F59E0B | Warning, attention, key insights |
| **Neutral** | Medium Gray | #D1D5DB | Section dividers, borders |
| **Surface** | Off-White | #FAFAFA | Card backgrounds, callout boxes |

**Color Philosophy:** Clean, neutral foundation with purposeful accent colors. Each accent has a semantic meaning. Avoid decorative color usage — every color serves an informational purpose.

---

## 3. Typography Rules

**Font Family:**
- **Headlines:** Bold display serif (Playfair Display, Georgia) or modern sans-serif (Inter, SF Pro)
- **Subheads:** Semi-bold sans-serif (Inter SemiBold, Roboto Medium)
- **Body:** Humanist sans-serif (Inter, Source Sans Pro)

**Hierarchy Table:**

| Role | Font | Size | Weight | Line Height | Letter Spacing | Notes |
|------|------|------|--------|-------------|----------------|-------|
| Hero Title | Display Serif/Sans | 48-64px | 700 | 1.1 | -0.01em | Strong visual presence |
| Section Head | Sans-serif | 32-40px | 600 | 1.2 | 0 | Clear hierarchy |
| Subhead | Sans-serif | 20-24px | 600 | 1.3 | 0 | Section headers |
| Body | Humanist Sans | 16px | 400 | 1.6 | 0 | Comfortable reading |
| Caption | Sans-serif | 12-14px | 400 | 1.4 | 0.01em | Metadata, labels |
| Pull Quote | Serif | 24-28px | 400 | 1.4 | 0 | Italic, accent color left border |
| Data Label | Sans-serif | 14px | 600 | 1.2 | 0.02em | Chart labels, annotations |

**Principles:**
- Editorial sophistication with clean letterforms
- Comfortable line height (1.6) for body text
- Clear hierarchy distinction between subheads and body
- Serif for headlines adds authority; sans-serif for body adds readability

## 4. Component Stylings

### Buttons
- **Primary:** Editorial Blue (#2563EB) background, white text, 6px border-radius, padding 12px 24px
- **Secondary:** White background, #1A1A1A border 1px, dark text, 6px border-radius
- **Text Link:** Editorial Blue, underline on hover, no background

### Cards & Containers
- White or Off-White (#FAFAFA) background
- 1px border (#D1D5DB) or subtle shadow (0 2px 8px rgba(0,0,0,0.06))
- 8-12px border-radius
- Padding 24-32px
- Callout boxes with left accent border (4px, accent color)

### Inputs & Forms
- Light gray background (#F8F9FA), 1px border (#D1D5DB)
- 6px border-radius
- Focus: Editorial Blue border, subtle shadow
- Labels: 14px, medium weight, dark gray

### Navigation
- Clean top bar, white background, bottom border
- Sans-serif, 14px, medium weight
- Active: Editorial Blue underline or text color

### Image Treatment
- Clean flat illustrations (not photos)
- Icon-based data visualization
- Visual metaphors for abstract concepts
- Consistent illustration style throughout

### Distinctive Components
- **Callout Box:** Light background with left accent border, contains key insight
- **Pull Quote:** Large italic text with left border in accent color
- **Flow Diagram:** Connected nodes with directional arrows, clean lines
- **Data Highlight:** Large number + label, accent color for the number
- **Section Divider:** Thin horizontal line (#D1D5DB) or subtle visual break

## 5. Layout Principles

**Spacing System:**
- Base unit: 8px
- Section padding: 48-64px vertical
- Content max-width: 1100px centered
- Card gap: 24px

**Grid & Container:**
- 12-column grid for complex layouts
- 2-3 column layouts for feature sections
- Single column for narrative flow sections

**Whitespace Philosophy:**
- Generous whitespace between sections
- Structured, organized spacing
- Information-dense but never cluttered
- Clear visual breaks between topics

**Border Radius Scale:**
- Small (cards, inputs): 6-8px
- Medium (callout boxes): 8-12px
- Large (feature containers): 12-16px

---

## 6. Depth & Elevation

| Level | Treatment | Use |
|-------|-----------|-----|
| 0 | Flat, no shadow | Base content, text |
| 1 | Subtle border | Cards, input fields |
| 2 | Light shadow (0 2px 8px rgba(0,0,0,0.06)) | Elevated cards, callout boxes |
| 3 | Medium shadow (0 4px 16px rgba(0,0,0,0.08)) | Modals, dropdowns, popovers |
| 4 | Heavy shadow (0 8px 32px rgba(0,0,0,0.12)) | Overlays, dialogs |

**Shadow Philosophy:** Shadows are subtle and functional, not decorative. They indicate elevation and hierarchy. Prefer borders over shadows for subtle separation.

---

## 7. Do's and Don'ts

### Do
- Create clear visual narrative flow
- Use structured multi-section layouts
- Include callout boxes for key insights
- Design visual metaphors for complex ideas
- Maintain magazine-quality polish
- Use clean flat illustrations (not photos)
- Label all data visualizations clearly

### Don't
- Use photographic imagery
- Create cluttered dense layouts
- Mix too many visual styles
- Add decorative elements without purpose
- Add slide numbers, footers, or logos
- Use inconsistent illustration styles
- Omit labels on charts and diagrams

---

## 8. Responsive Behavior

| Breakpoint | Width | Behavior |
|------------|-------|----------|
| Mobile | < 640px | Single column, stacked cards, headline 32px |
| Tablet | 640-1024px | 2-column grid, headline 40px, maintained structure |
| Desktop | > 1024px | Full multi-column layout, headline 48-64px |

**Touch Targets:** Minimum 44px for interactive elements

**Collapsing Strategy:**
- Multi-column grids collapse to single column
- Side-by-side text + illustration stacks vertically
- Callout boxes remain full-width
- Flow diagrams simplify to vertical stacks

**Image Behavior:**
- Illustrations scale proportionally
- Data visualizations reflow to fit container
- Maintain readability of all labels at all sizes

## 9. Agent Prompt Guide

### Quick Color Reference
- Background: #FFFFFF (White) / #F8F9FA (Light Gray)
- Primary Text: #1A1A1A (Near Black)
- Accent: #2563EB (Editorial Blue), #F97316 (Coral)
- Divider: #D1D5DB (Medium Gray)

### Example Component Prompts

1. **Explainer Section:** "White background with a 2-column layout. Left: clean flat illustration of a concept. Right: section title in bold serif (32px), body text in sans-serif (16px, line-height 1.6), and a callout box with left blue (#2563EB) border containing a key insight."

2. **Data Flow Diagram:** "Light gray (#F8F9FA) background with a horizontal flow diagram. 4 connected nodes with icons, connected by arrows. Each node has a title and short description. Editorial Blue (#2563EB) for active nodes, gray for inactive."

3. **Pull Quote Highlight:** "Full-width section with off-white (#FAFAFA) background. Large italic pull quote (24px serif) with left border in coral (#F97316). Attribution below in 14px dark gray."

4. **Feature Grid:** "3-column grid on white background. Each column has an icon illustration, bold title (20px), and body text (16px). Cards have subtle border (#D1D5DB) and 8px border-radius. Equal height cards."

5. **Process Steps:** "Numbered step sequence (01-04) on white background. Each step has a large number in Editorial Blue, a title, and description. Steps connected by a thin vertical line. Alternating left/right layout."

6. **Comparison Layout:** "Split layout comparing two concepts. Left side: light blue tint background with concept A details. Right side: light coral tint background with concept B details. Center divider line."

### Iteration Guide

1. **Start with structure:** Define the information hierarchy before adding visual elements.
2. **One concept per section:** Each section should communicate one clear idea.
3. **Visual metaphors:** Replace abstract concepts with concrete visual representations.
4. **Consistent illustrations:** Maintain the same illustration style throughout.
5. **Label everything:** Charts, diagrams, and data visualizations must have clear labels.
6. **Readability first:** Ensure body text is always comfortable to read at 16px.
